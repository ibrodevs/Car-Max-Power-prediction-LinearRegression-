## О проекте
В репозитории есть два основных скрипта:
- `train.py` — обучение модели и сохранение артефактов;
- `predict.py` — интерактивный CLI для ручного ввода признаков и получения прогноза.

Модель: `LinearRegression` из `scikit-learn`.

Целевая переменная:
- `Max Power_num` (числовое значение мощности, извлечённое из текстового столбца `Max Power`).

## Что умеет проект
- Загружать CSV-данные с характеристиками авто.
- Преобразовывать текстовые поля (`Engine`, `Max Power`, `Max Torque`) в числовые признаки.
- Обучать линейную регрессию на наборе признаков.
- Вычислять `R2` на тестовой выборке.
- Сохранять обученную модель в `model.pkl`.
- Строить и сохранять график `actual_vs_predicted.png`.
- Запускать интерактивные предсказания по одному объекту через терминал.

## Подробный процесс обучения
В `train.py` выполняются шаги:

1. Загрузка данных:
   - `pd.read_csv("car details v4.csv")`

2. Преобразование текстовых столбцов в числа:
   - функция `num(x)` через regex извлекает первое числовое значение из строки;
   - для столбцов `Engine`, `Max Power`, `Max Torque` создаются новые признаки:
     - `Engine_num`
     - `Max Power_num`
     - `Max Torque_num`

3. Выбор признаков и целевой переменной:
   - используется фиксированный список колонок;
   - строки с пропусками удаляются через `.dropna()`.

4. Деление на train/test:
   - `test_size=0.2`
   - `random_state=2`

5. Обучение модели:
   - `LinearRegression().fit(X_train, y_train)`

6. Сохранение артефакта:
   - в `model.pkl` сохраняются:
     - `model` (обученный объект модели)
     - `feature_names` (порядок признаков)

7. Оценка качества:
   - предсказания на `X_test`;
   - расчёт `r2_score`;
   - вывод процента в консоль.

8. Визуализация:
   - scatter-график фактических vs предсказанных значений;
   - красная линия `y=x` как эталон идеального предсказания;
   - сохранение в `actual_vs_predicted.png`.

## Подробный процесс предсказания
`predict.py` работает в интерактивном режиме:

1. Пытается открыть `model.pkl`.
2. Если файла нет — просит сначала запустить `train.py`.
3. Берёт `feature_names` из файла модели.
4. По очереди запрашивает каждое поле у пользователя.
5. Проверяет, что ввод — число (`read_float`).
6. Формирует `DataFrame` в правильном порядке признаков.
7. Вызывает `model.predict(...)`.
8. Печатает результат с округлением до 2 знаков.
9. Спрашивает, хотите ли ещё предсказание (`yes/no`).

Важно: ответы на вопрос о продолжении нужно вводить как `yes/y` или `no/n`.

## Описание входных признаков
При обучении используются такие признаки (X):
- `Price`
- `Year`
- `Kilometer`
- `Length`
- `Width`
- `Height`
- `Seating Capacity`
- `Fuel Tank Capacity`
- `Engine_num`
- `Max Torque_num`

Цель (y):
- `Max Power_num`

### Примечание про единицы измерения
Поскольку числа извлекаются regex-ом из строк, единицы измерения (например, `CC`, `bhp`, `Nm`) не учитываются напрямую как отдельные категориальные признаки. Важно, чтобы формат данных был согласованным внутри датасета.

## Выходные артефакты
После `python train.py`:

- консольный вывод `R2 (accuracy): ...%`.

## Проверка качества модели
Используемая метрика:
- `R²` (`r2_score`) на отложенной тестовой выборке.

Интерпретация:
- ближе к `1.0` (или `100%` в выводе скрипта) — лучше;
- около `0` — модель почти не объясняет вариативность цели;
- ниже `0` — модель хуже базового предсказания средним.

## Типичные ошибки и решения
### 1) `model.pkl not found`
Причина: модель ещё не обучена или файл удалён.
Решение: запустите `python train.py`.

### 2) `ModuleNotFoundError`
Причина: зависимости не установлены.
Решение:
```bash
pip install -r requirements.txt
```

### 3) Ошибка чтения CSV
Причины:
- неверное имя файла;
- запуск из другой директории.

Решение:
- убедитесь, что файл называется точно `car details v4.csv`;
- запускайте команды из корня проекта.

### 4) Некорректный ввод в `predict.py`
Причина: введён текст вместо числа.
Решение: введите числовое значение (например, `2019`, `1500000`, `120.5`).

### 5) Пустые/неполные данные после `dropna()`
Причина: много пропусков в критичных колонках.
Решение:
- проверить качество исходного CSV;
- при необходимости добавить обработку пропусков вместо полного удаления строк.
